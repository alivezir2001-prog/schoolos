# Purpose

The SchoolOS UX Blueprint defines how educators, learners, parents and administrators interact with SchoolOS.

Rather than describing software screens, it describes educational journeys.

Every screen, workflow and AI capability exists to support meaningful educational experiences.

This document serves as the primary user experience architecture for SchoolOS.

---

# Vision

SchoolOS is designed around educational experiences rather than software modules.

Users should never wonder where to go next.

Every interaction should naturally lead to the next educational action.

Technology should disappear.

Educational understanding should remain.

---

# Educational Philosophy

SchoolOS is built upon four educational actions.

Observe.

Understand.

Support.

Grow.

Every workflow within SchoolOS strengthens one or more of these actions.
