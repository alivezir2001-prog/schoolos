# ADR-014

## Title

Repository Is The Permanent Project Memory

---

## Status

Accepted

---

## Date

2026-07-13

---

## Context

Development conversations are temporary.

Architectural knowledge should remain available independently of any individual discussion.

Long-term projects require persistent documentation.

---

## Decision

The repository becomes the permanent memory of the project.

Architectural knowledge is preserved through documentation, ADRs and version control rather than conversational history.

---

## Rationale

Knowledge should outlive conversations.

Documentation provides continuity, traceability and collaborative development.

---

## Consequences

Architectural decisions are documented as ADRs.

Theories are documented separately.

Repository documentation becomes the single source of truth.

---

## Alternatives Considered

### Preserve knowledge inside conversations.

Rejected because conversations are temporary and difficult to reference.
